Internet Messaging Access Protocol

What is it?

IMAP (Internet Message Access Protocol) is one of the world’s most widely used communication protocols, supporting millions of users globally. It enables modern email synchronization across multiple devices in real time. IMAP stores messages on the server, allowing users to manage their emails seamlessly from anywhere. Major providers like Google, Yahoo, and Outlook use IMAP to ensure mobile-friendly and scalable email access.

Features and Functionality

• IMAP establishes a connection to the mail server when users log in.

• Username and passwords are verified, and messages are downloaded only when users access specific folders like Inbox or Outbox.

• Emails stay on the server and remain synchronized until the user deletes them.

HISTORY

IMAP was created by Mark Crispin at Stanford in the 1980s (Nylas, n.d.). The specification of IMAP origniates in a form of a "Request for Comments" (RFC), s memo describing how to efficiently implement the protocol. When Crispin designed IMAP, he intended to be better than POP3 due to the real-time incoming messages and response.

IMAP Versions

IMAPv3: enables real time accessibility for users and email synchronization for multiple devices.

IMAP2bis: an extension of IMAP2, supports MIME body structurres and managing additional functionalities.

IMAP4: Most recent, deployed in 1998, intened to cover all disadvantages that POP3 displayed.

Advantages

• Provides synchronization between user sessions

• Allows accessibility from all capable remote devices

• Able to store content without the requirements of physical storage space
